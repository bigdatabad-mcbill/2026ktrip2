# CloseTrip MVP 개발 계획

## 목적
`2026ktrip2` 리포지토리는 CloseTrip 여행 서비스의 개발 출발점입니다. README에 작성된 UX/기술/사업 요구사항을 바탕으로, 우선 MVP 수준의 웹 프로토타입을 구현하여 핵심 사용자 흐름을 검증합니다.

## 선택 기술 스택
- Frontend: React + Vite + TypeScript
- UI: CSS 기반 단일 페이지 앱
- 목표: 웹에서 빠르게 동작하는 프로토타입으로 일정, 코디, 포토스폿, 다이어리, 설정 화면 구성

## MVP 범위
- 언어 선택 및 다국어 중 한국어/영어 우선 지원
- 홈 화면에서 오늘의 여행 요약 제공
- 코디 추천 카드 및 추천 근거 표시
- 포토스폿 화면의 장소 리스트 및 조건 필터
- 자동 다이어리 요약 화면
- 설정 화면에서 언어/단위 표시
- 탭 네비게이션으로 흐름 구성

## 개발 단계
1. 프로젝트 스켈레톤 구성
2. 공통 UI 컴포넌트 및 mock 데이터 작성
3. Home / Style / Spot / Diary / Settings 화면 구현
4. CSS 스타일 적용 및 반응형 간격 조정
5. 문서 정리 및 다음 확장 계획 작성

## 확장 계획
- 이후 React Native 또는 Flutter로 모바일 앱 이전
- 백엔드 API 연결, 위치/날씨/장소 데이터 연동
- 실제 다국어 번역 리소스 및 권한 흐름 구현
- AI 다이어리 생성 및 추천 엔진 추가
